# «Я Бао Завари» — frontend-сайт

Статический многостраничный frontend, готовый для публикации на GitHub Pages. Проект развивает утверждённый hero с Full HD-видео и реализует структуру из ТЗ: церемонии, пространства, меню, события, сертификаты, контакты, бронирование, FAQ и юридические страницы.

## Быстрый запуск

1. Распакуйте архив.
2. Загрузите **содержимое папки** в корень репозитория GitHub.
3. В GitHub откройте `Settings → Pages`.
4. Выберите `Deploy from a branch`, ветку `main`, каталог `/root`.
5. Главная страница — `index.html`.

Локально проект лучше запускать через сервер:

```bash
python -m http.server 8000
```

Затем открыть `http://localhost:8000`.

## Страницы

- `index.html` — главная;
- `ceremonies.html` — форматы церемоний;
- `spaces.html` — VIP-комнаты и пространства;
- `menu.html` — меню и фильтрация;
- `events.html` — афиша и фильтрация;
- `certificates.html` — подарочные сертификаты;
- `about.html` — о чайной;
- `contacts.html` — контакты;
- `booking.html` — отдельная форма бронирования;
- `privacy.html`, `consent.html` — юридические шаблоны;
- `404.html` — страница ошибки.

## Что уже работает

- адаптивная шапка и полноэкранное мобильное меню;
- Full HD hero-видео 1920×1080 с WebP-постером;
- остановка видео вне экрана, в неактивной вкладке и при `prefers-reduced-motion`/`save-data`;
- модальное бронирование с передачей выбранной церемонии, пространства или события;
- маска телефона и клиентская валидация;
- фильтры меню и событий;
- FAQ-аккордеон;
- галерея с lightbox;
- адаптивы для desktop, laptop, tablet, mobile и экранов до 390 px;
- уникальные title, description и H1;
- Open Graph, LocalBusiness/CafeOrCoffeeShop Schema, Sitemap и robots.txt;
- видимые focus-состояния и базовая клавиатурная доступность.

## Что необходимо заменить перед публикацией

Элементы с визуальной меткой **«шаблон — уточнить»** содержат неподтверждённые данные.

Обязательно уточнить:

1. домен вместо `https://example.ru` в canonical, Open Graph, `sitemap.xml` и `robots.txt`;
2. единое написание адреса: в 2ГИС — `Кирова, 94`, в Яндекс Картах — `Кирова, 94А`;
3. названия, продолжительность, вместимость, цены и состав церемоний;
4. фактические названия и вместимость комнат;
5. условия предоплаты, отмены и переноса;
6. реальные даты, цены и количество мест на события;
7. номиналы и правила подарочных сертификатов;
8. юридические реквизиты и e-mail оператора персональных данных;
9. реальные фотографии комнат, фасада, команды, событий и входа.

В проекте используются созданные для макета изображения-заглушки. Hero-видео оставлено оригинальным. Публичные фотографии из карт и VK автоматически не копировались: их следует добавить только при наличии права на использование и заменить файлы в `assets/images/` с сохранением названий либо обновить пути в HTML.

## Формы

Формы работают в **демонстрационном режиме**: валидируют данные и показывают успешное состояние, но не отправляют заявку на сервер.

Для production подключите один из вариантов:

- WordPress AJAX + сохранение в базе;
- Contact Form 7 / Fluent Forms;
- собственный PHP-обработчик;
- CRM webhook;
- Telegram Bot API через серверный обработчик.

Не отправляйте Telegram Bot Token или SMTP-пароль из frontend JavaScript.

## Аналитика

Добавьте Яндекс Метрику перед закрывающим `</head>` и настройте события:

- `open_booking_form`;
- `submit_booking_form`;
- `submit_booking_success`;
- `click_phone`;
- `click_map`;
- `click_messenger`;
- `click_certificate`;
- `click_event`;
- `ceremony_select`;
- `room_select`;
- `scroll_50`;
- `scroll_90`.

Рекомендуемые параметры: `selected_ceremony`, `selected_room`, `selected_event`, `guests_count`, `page_type`, `traffic_source`.

## Переход на WordPress

Для CMS рекомендуется собственная тема WordPress + ACF Pro:

- CPT `ceremony`;
- CPT `space`;
- CPT `event`;
- CPT `certificate`;
- CPT `review`;
- CPT `faq`;
- опционально CPT `tea_item`.

Повторяющиеся секции следует перенести в `template-parts`, а данные карточек — в ACF-поля. Не переносите всю разметку в один шаблон или `functions.php`.

## Источники фактических данных

- 2ГИС: адрес, телефон, режим работы, рейтинг, ссылка на VK;
- Яндекс Карты: телефон, часы, особенности, товары и услуги, отзывы;
- позиционирование и ТЗ заказчика;
- фотографии и события в текущей сборке частично являются шаблонными.
